# 阅读目标模式

根据用户目的调整输出重点。

## 判断是否值得读

优先输出：

- 一句话结论
- 和用户方向的相关性
- 方法新意
- 实验可信度
- 阅读优先级

减少：

- 过长方法细节
- 逐段翻译

## 组会汇报

优先输出：

- 开场问题
- 方法主线
- 关键图表
- 听众容易问的问题
- 讨论点

建议结构：

1. 这个问题为什么重要
2. 旧方法哪里不够
3. 这篇论文的核心 idea
4. 方法怎么实现
5. 实验是否支撑
6. 局限和讨论

## Related Work

优先输出：

- 这篇论文属于哪条技术线
- 它解决了哪个前人工作没解决的问题
- 它和相邻方法的差别
- 可以如何引用

避免：

- 宣传式评价
- 没有比较对象的“首次提出”
- 不确定的历史定位

## 复现前阅读

优先输出：

- 最小复现目标
- 数据集和指标
- baseline
- 代码入口
- 风险点
- 实验记录模板

关注：

- 论文描述和代码实现是否一致
- 是否依赖旧 API、旧数据或不可用 checkpoint
- 评价指标是否容易复现
